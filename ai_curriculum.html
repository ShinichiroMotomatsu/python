    研修の目的

    「使ってみた」で止めず、マネジメント業務の 意思決定・育成・効率化 にAIを組み込むことをゴールとする。

    研修ゴール(受講後の状態)

    • 自分の業務:議事録要約・メール起案・資料構成案・データ整理を、Gemini連携機能で日常的に回せる
    • 1on1とフィードバック:メンバーの状況と事実を渡せば、論点・問い・コメント素案がAIから引き出せる
    • 組織マネジメント:チーム課題・スキルマップ・育成プランをAIと壁打ちしながら構造化できる
    • 判断力:AI出力をそのまま使う/手直しする/捨てる の判断軸を言語化できる

    受講者像の Before / After

    「ChatGPTを触ったことはあるが、業務に組み込めていない」管理職が典型。

    Before — ありがちな状態

    • 会議のたびに自分で議事録を書いている
    • 1on1の準備に毎回30分〜1h消費
    • 評価コメントが「印象」ベースで似た文章になる
    • AIに何度か触ったが、出力が浅くて使えなかった
    • セキュリティが不安で、業務情報を入れられない

    After — 研修後の状態

    • Google Meetの自動文字起こしからタスク抽出が標準フロー
    • 1on1準備はメンバー別Gemにテンプレ化(15分以内)
    • 評価コメントを「事実→評価→期待」の3層で整える
    • 壁打ちの3モードを使い分け、深い出力を引き出せる
    • 入れていい情報/ダメな情報の判断軸を持っている

    →

    • 全体タイムキープと節目の解説

    MODULE 1

    AI基礎リテラシー

    LLMの仕組みを「ざっくり正しく」理解し、得意/不得意・ハルシネーション・セキュリティを押さえる(60分)

    M1LLM は「確率的に次の単語を選ぶ」モデル

    完璧な知識ベースではなく、入力に対してもっともらしい続きを生成する装置。

    3つの含意

    • 同じ入力でも出力が揺れる — 確率モデルだから。重要な判断は複数回生成して見比べる
    • 知らないことも自然に答える — 「らしさ」を優先するため、事実と異なる出力(ハルシネーション)が起きる
    • 入力の質が出力の質を決める — 文脈・制約・出力形式を渡すほど、確率の山が「狙った方向」に寄る

    M1AIの得意/不得意マップ

    「任せていい仕事」と「人が判断すべき仕事」を見分ける目を持つ。

    M1ハルシネーション対策 — 検証3原則

    「AIが堂々と嘘をつく」現象は、原理上ゼロにできない。検証を仕組み化する。

    ① 出典分離

    「事実」と「AIの解釈」を必ず分ける。
    プロンプトに「事実部分には [F]、解釈には [I] を付けて出力して」と指示。

    ② 複数生成

    重要な判断は同じ問いを2〜3回聞き、差分を見る。
    表現が安定しない部分は不確実性が高い。

    ③ 反証要求

    「この出力に対する3つの反論を挙げて」と続けて指示。
    賛同しか返ってこないAIは要注意。

    高リスクな質問パターン

    特に幻覚を起こしやすい

    • 具体的な数値・統計を聞く(「日本のSaaS市場は何兆円?」)
    • 固有名詞のスペル・URL・引用元
    • 過去の特定の人物・出来事の詳細
    • 知識カットオフ以降の最新情報

    比較的安全な使い方

    • こちらが渡したテキストの要約・整形・分類
    • 論理構造のチェック・抜け漏れ指摘
    • 表現の言い換え・トーン調整
    • こちらの考えに対する批判・反論の生成

    M1情報セキュリティの判断軸

    「会社が法人契約しているGemini」は学習に使われない設定。とはいえ「何でもOK」ではない。

    分類具体例原則
    ① 公開情報プレスリリース・公開Webコンテンツ・自社サービスサイトの文言入力OK。出力結果も共有可。
    ② 社内一般情報会議のアジェンダ・自分のメモ・社内向けドキュメントの構成業務契約のGeminiにのみ入力可。出力は社外共有時に再確認。
    ③ 社外秘・機密未公開財務情報・契約書・顧客固有情報・採用候補者の個人情報原則入力しない。やむを得ない場合は固有名詞・数値を伏字化。
    ④ 個人情報・人事情報メンバーの実名・評価・健康・給与情報必ず仮名化。「メンバーA(入社3年目・主任)」のように。
    1on1・人事系のハンズオン(M4)では、必ずメンバー名を伏字(A/B/Cやイニシャル)で進めてください。
    判断に迷ったら:「これが万一そのまま外に出ても問題ないか?」を自問。問題があるなら、抽象化または固有情報を抜く。
    MODULE 2

    プロンプトの型と
    壁打ち作法

    5要素の型と、壁打ちの3モード(発散・構造化・批判)を使い分ける(75分)

    M2良いプロンプトの5要素

    「役割 / 背景 / 依頼 / 制約 / 出力形式」 — どれか1つを意識的に厚くするだけで、出力品質は跳ね上がる。

    要素役割例
    ① 役割
    (Role)    		AIの立ち位置を固定する「SaaS企業のVP of Sales として」
    ② 背景
    (Context)    		状況・前提を共有する「ARR 10億円、CSが弱い」
    ③ 依頼
    (Task)    		具体的に何をしてほしいか「来期のKPIツリーを起案」
    ④ 制約
    (Constraints)    		条件・観点・除外事項「販管費は据置・営業人員追加なし」
    ⑤ 出力形式
    (Format)    		長さ・構造・トーン「マークダウン表で、3案併記」

    Before / After 例

    Before

    「来期のKPIを考えて」

    After

    「あなたは SaaS企業のVP of Sales。当社はARR10億円・CS弱め。来期のKPIツリーを3案起案して。条件:営業人員追加なし。マークダウン表形式で、各案の狙い・リスクも併記」

    覚え方:「ヤク(役割)ハイ(背景)にイ(依頼)っぱいセ(制約)ってシュ(出力)」と語呂で覚える。

    M2壁打ちの3モード

    同じテーマでも、AIに依頼するモードを切り替えることで引き出せる価値が変わる。

    ① 発散モード(Diverge)

    目的:選択肢を増やす
    合図ワード:「○つ案を挙げて」「角度を変えて」「ありえない案も含めて」
    使う場面:戦略の初期検討、施策のブレスト、1on1の問いの洗い出し

    ② 構造化モード(Structure)

    目的:情報を整理する
    合図ワード:「分類して」「フレームに当てはめて」「優先順位を付けて」
    使う場面:会議メモの構造化、課題の整理、評価コメントの3層化

    ③ 批判モード(Critique)

    目的:自分の案を叩く
    合図ワード:「反論を3つ」「最も致命的な弱点は?」「経営会議で質問されそうな点」
    使う場面:提案前の最終チェック、意思決定のリスク洗い出し

    使い分けの黄金パターン

    1. 発散選択肢を10案出す
    →
    2. 構造化3〜5案に絞り、比較表に
    →
    3. 批判選んだ案への反論を引き出す
    →
    4. 人間が決める批判を踏まえて自分の意思で判断
    注意:AIは依頼者に同調しがち。批判モードを必ず1回挟むだけで、意思決定の質が変わります。

    M2ハンズオン① — 自分の業務課題で3モード壁打ち

    所要45分 / Gemini(gemini.google.com もしくはWorkspaceサイドバー)で実施

    手順

    1. 下のプロンプトをコピーしてGeminiに貼り付け、【 】の中を自分の状況で埋める
    2. 送信後、AIの指示に従って対話を進める(発散→構造化→批判の順)
    3. 最後にAIに「振り返りサマリを出して」と依頼し、ワークシートに転記

    初期プロンプト(コピペ用)

    # 役割 あなたは私の「思考の壁打ち相手」です。私の業務課題に対して、3つのモードを順に使って、深い検討を引き出してください。 # 私の状況 - 役割: 【SaaS企業の○○部 マネージャー、部下10名】 - 今困っていること: 【例: チームの新規商談数が3ヶ月連続で目標未達。原因を整理して打ち手を考えたい】 - 制約: 【例: 増員不可、追加予算なし、3ヶ月以内に改善着手】 # 進め方 【モード1:発散】 まず「考えられる原因」を、視点を変えて10個挙げてください。 ありえなさそうな仮説も含めて構いません。 【モード2:構造化】 次に、私が選んだ仮説(私が番号で答えます)を起点に、 「原因の構造」をロジックツリー形式で2階層分解してください。 【モード3:批判】 最後に、私が考えた打ち手(私が文章で書きます)について、 「経営会議で出そうな3つの反論」を出してください。 # 最後 すべて終わったら、以下のフォーマットで振り返りサマリを出力してください。 - 今日扱った課題: - 採用した仮説: - 採用した打ち手: - 残った論点(次回考えるべきこと): それでは、まず【モード1】からお願いします。

    気づき

    壁打ち中に「自分一人では出なかった視点」を1つ書く

    明日試すこと

    この壁打ちで決まった、具体的な次の一手

    AIに任せきれなかった部分

    最終的に自分の判断で覆した/補完した点

    M3ハンズオン②-A : 議事録から要約・タスク抽出

    所要15分 / 題材は事前配布の「サンプル会議トランスクリプト」 or 自分のMeet録画文字起こし

    手順

    1. Google Docs に文字起こしテキストを貼り付ける(または該当Docを開く)
    2. 右側の「Geminiに質問」サイドバーを開き、下のプロンプトを貼る
    3. 出力を確認 → 不足があれば追記指示

    プロンプト

    # 役割 あなたは経験豊富な議事録ライターです。 # 入力 このドキュメントの内容を題材とします。 # 依頼 以下の4ブロックに分けて出力してください。 1. 会議のサマリ(3行以内、何のための会議で何が決まったか) 2. 決定事項(箇条書き、必ず「誰が決めたか」を併記) 3. ToDo(担当者 / 期限 / 内容 の表形式。期限が明示されていない場合は「未定」と記載) 4. 論点・未解決事項(次回持ち越しの議題) # 制約 - 発言者が特定できない部分は「不明」と書くこと - 推測で補完しない(推測が必要な場合は明示) - 出力はマークダウン形式
    応用:続けて「この会議で言われなかったが、本来詰めるべきだった論点を3つ指摘して」と聞くと、抜け漏れチェックに使えます。

    M3ハンズオン②-B : メールの下書き・トーン調整
