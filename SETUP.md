# AI アセスメント — 実行環境セットアップマニュアル

このディレクトリのファイルを Google Apps Script(以下 GAS)に配置して、
AI 利活用アセスメントの **受験 Web アプリ** と **ダッシュボード** を社内に
公開するための手順書です。

## 1. 前提

| 項目 | 内容 |
|---|---|
| Google Workspace アカウント | 本スクリプトをデプロイする実行者用(個人 Gmail 不可) |
| Google ドライブ | スプレッドシートが自動作成される置き場 |
| Gemini API キー | [aistudio.google.com](https://aistudio.google.com/) で発行(無料枠で動作確認可) |
| ブラウザ | Chrome / Edge / Safari 推奨 |

> **重要**: Web アプリのデプロイは「会社ドメイン内」に絞ることを推奨。
> 必要に応じて Apps Script のデプロイ設定で「アクセスできるユーザー」を変更。

---

## 2. ファイル構成

`gas/` ディレクトリ内の以下 4 ファイルを GAS プロジェクトに配置します。

| ファイル | GAS 上のファイル種別 | GAS 上の名前 |
|---|---|---|
| `Code.gs` | スクリプト | `Code` |
| `appsscript.json` | マニフェスト(`プロジェクトの設定` で「`appsscript.json` マニフェスト ファイルをエディタで表示する」を ON にすると編集できる) | `appsscript` |
| `index.html` | HTML | `index` |
| `dashboard.html` | HTML | `dashboard` |

---

## 3. 初期セットアップ手順

### 3-1. GAS プロジェクトの作成

1. [script.google.com](https://script.google.com) を開く
2. 左上「**新しいプロジェクト**」
3. プロジェクト名を「**AI アセスメント**」など分かりやすい名称に変更

### 3-2. ファイルの配置

1. 左ペインの **+** ボタンから:
   - 「**スクリプト**」を選び `Code` を作成 → 中身を `Code.gs` の内容で置き換え
   - 「**HTML**」を選び `index` を作成 → 中身を `index.html` の内容で置き換え
   - 「**HTML**」を選び `dashboard` を作成 → 中身を `dashboard.html` の内容で置き換え
2. プロジェクト設定:
   - 左サイドバー ⚙️ **プロジェクトの設定**
   - 「**`appsscript.json` マニフェスト ファイルをエディタで表示する**」を **ON**
3. 左ペインの `appsscript.json` を開き、中身を本ディレクトリの `appsscript.json` の内容で置き換え
4. すべて **Ctrl + S**(Mac: **Cmd + S**)で保存

### 3-3. Gemini API キーを登録

1. 左サイドバー ⚙️ **プロジェクトの設定**
2. 下部「**スクリプト プロパティ**」セクション
3. **スクリプト プロパティを追加** をクリック
4. 入力:
   - **プロパティ**: `GEMINI_API_KEY`
   - **値**: aistudio.google.com で発行した API キー
5. **スクリプト プロパティを保存**

### 3-4. スプレッドシートの初期化

1. スクリプトエディタの上部「**関数を選択**」プルダウン → **`setupSpreadsheet`**
2. **▷ 実行** をクリック
3. 初回は権限承認ダイアログが出るので:
   - 「**権限を確認**」 → 自分のアカウントを選択
   - 「**詳細**」→「**…(プロジェクト名)に移動(安全ではないページ)**」(社内ドメイン外公開する場合は警告が出るが、社内利用なので問題なし)
   - スコープを確認して「**許可**」
4. 「**実行ログ**」(画面下 or `表示 > 実行ログ`)で `結果保存シートを準備しました: https://...` の URL が出る
5. その URL を開いて、**Results** タブが空のシートとして作られていることを確認(ヘッダ 24 列)

### 3-5. Gemini 接続テスト(任意)

1. 関数選択 → **`testGeminiConnection`** → ▷ 実行
2. 実行ログに `HTTP 200: {"candidates": [...]}` のような出力が出ればOK
3. もし `HTTP 4xx` / `HTTP 503` が出る場合は、API キーや有料アカウントの状態を確認

---

## 4. Web アプリのデプロイ

1. 右上 **デプロイ** → **新しいデプロイ**
2. 種類選択(歯車アイコン)→ **ウェブアプリ**
3. 設定:
   - **説明**: 「初回デプロイ」など任意
   - **ウェブアプリの実行者**: **自分**
   - **アクセスできるユーザー**: 「**自社ドメイン全員**」推奨(社内のみ受験可能にする)
4. **デプロイ** をクリック
5. 表示される **ウェブアプリ URL** をコピー(例: `https://script.google.com/macros/s/AKfycb.../exec`)

### 4-1. URL の使い分け

| URL | 用途 |
|---|---|
| `<上記URL>` | **受験用**(一般受験者に共有) |
| `<上記URL>?page=dashboard` | **ダッシュボード用**(管理者用、面談補助資料の参照) |

> ダッシュボードを管理者だけに見せたい場合は、`Code.gs` の `ADMIN_EMAILS = [...]` に管理者の Google アカウント(メアド)を列挙すると、それ以外は閲覧不可になります。

---

## 5. 動作確認

### 5-1. 受験フロー

1. 受験 URL を開く
2. 動作モード(**昇格試験 / 管理職オンボーディング**)、氏名、所属、職種(大分類 → 詳細)を入力
3. **テスト開始** をクリック
4. タイマー 45 分が起動。問題に回答していく
5. 最後の Section E(自由記述)まで進めて **提出**
6. 採点中ローディング(20〜60 秒)→ 結果画面に総合スコアと AI コメントが表示される

### 5-2. ダッシュボード

1. `<URL>?page=dashboard` を開く
2. 右上のバージョンバッジが **緑(同期済み)** であることを確認
3. 受験者一覧から「**詳細**」ボタン → モーダルに受験詳細 + AI コメントが表示される

## 6. 運用上の注意

### 6-1. コードを更新したとき(重要)

スクリプトエディタで `Code.gs` や `index.html`、`dashboard.html` を編集して **Ctrl+S** で保存しただけでは、配布中の URL には反映されません。**必ず以下を実施**してください:

1. スクリプトエディタ右上 **デプロイ** → **デプロイの管理**
2. 該当デプロイの ✏️ **編集**
3. **バージョン** プルダウン → **新バージョン** を選択
4. **デプロイ** をクリック

これをやらないと、エディタは最新でも Web アプリ URL は古いコードを配信し続けます。ダッシュボード右上の **バージョンバッジが赤** になっていたら未デプロイのサインです。

### 6-2. シートが「壊れた」と感じたら

スキーマ変更を跨いで運用していると、ヘッダと実データの位置がズレることがあります(列ズレ)。その場合:

1. スクリプトエディタで関数選択 → **`migrateSheetToCanonicalSchema`** → ▷ 実行
2. 元のシートが `Results_backup_yyyyMMdd_HHmmss` にリネームされ、新しい正規化 `Results` シートが作られる
3. 実行ログで内訳(21 列/19 列/24 列の判別件数)を確認

旧データの「モード/職種/AI 活用記述/AI 評価アドバイス」列は空欄になります(当時記録していないため復元不能)。

### 6-3. ダッシュボードの詳細が開かないとき

スクリプトエディタで `debugSubmissionDetailLookup` を実行し、実行ログを確認:

- **オブジェクトが返る**: サーバ側健全 → 再デプロイ + 強制リロード(Cmd+Shift+R / Ctrl+Shift+R)
- **null / undefined / 例外**: ログ全文を共有して相談

### 6-4. Gemini API の枠管理

- 提出 1 回につき Gemini API を **2 回**呼び出す(D 採点 + 総合アドバイス生成)
- 無料枠は低 RPM。受験者が同時多発するとエラーになる可能性あり
- 規模が大きい場合は有料枠 (Google Cloud Project へ紐付け)を検討

## 7. デフォルト設定値の調整

`Code.gs` 冒頭の定数で挙動を調整できます:

| 定数 | デフォルト | 説明 |
|---|---|---|
| `GEMINI_MODELS` | `['gemini-2.5-flash', 'gemini-2.5-flash-lite', 'gemini-1.5-flash']` | 上から順にフォールバック |
| `GEMINI_MAX_ATTEMPTS` | `4` | 各モデルへのリトライ回数 |
| `PASS_PCT` | `0.60` | 及第点ライン(総合 60%) |
| `EXCELLENT_PCT` | `0.80` | 優良ライン(総合 80%) |
| `D_MIN_PCT` | `0.50` | セクション D の最低達成率(これ未満なら自動的に「要努力」) |
| `ADMIN_EMAILS` | `[]` | ダッシュボードを閲覧できる管理者のメアド配列。空なら Web アプリのアクセス権限に従う |

`index.html` の `TIMER_TOTAL_SEC` で試験時間(秒)を変更できます。

## 8. ファイル更新時のバージョン管理

`Code.gs` と `dashboard.html` には ビルド ID を埋め込んでいます。

- `Code.gs` の `SERVER_BUILD` 定数
- `dashboard.html` の `CLIENT_BUILD` 定数

両者は **必ず同じ値**にして、デプロイのたびに更新します。
ダッシュボードを開くと右上に `DASH: xxx / SRV: yyy` のバッジが出るので、両者が一致(緑)していればデプロイ反映済みです。

## 9. トラブルシューティング

| 症状 | 原因の候補 | 対処 |
|---|---|---|
| 受験 URL を開くと真っ白 | デプロイがされていない / URL が間違っている | デプロイの管理で URL 再確認 |
| 提出時に Gemini エラー | API キー未設定 / 503 一時障害 | スクリプトプロパティ確認、数分後に再提出 |
| ダッシュボードに「読み込み中…」のまま | Web アプリの権限不足 | デプロイ設定の「アクセスできるユーザー」確認 |
| ダッシュボードのバッジが赤 | Code.gs の最新が未デプロイ | 6-1 の手順で新バージョン発行 |
| 「詳細」ボタンで空応答 | 上記 + ブラウザキャッシュ | 強制リロード(Cmd/Ctrl+Shift+R) |
| カテゴリ判定が想定と違う | スコア閾値の調整必要 | 第 7 章の定数を調整 |
