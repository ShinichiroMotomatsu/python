1. OVERVIEW

プロジェクト概要

  • 作るもの: 中世ヨーロッパ風・剣と魔法のオープンワールドRPG。中学時代(1980年代)に夢見たゲームを、いまの技術と人生経験で作る
  • 体験の核: 未知のフィールドに踏み込むわくわく＋「既知の再発見」。同じ場所を「過去(ファンタジー)」と「現在(2026年の実在の街)」の二つの時代で探索する
  • プレイ環境: iPhoneなどスマホ縦持ち(PCでも矢印キー+Enterで可)
  • 作者の実像との対応: 多摩川沿いの原風景、麻布台ヒルズ(再開発)、変わらない東京タワー、消えたものたち(公衆電話・レンタルビデオ等)をファンタジーに翻訳して織り込む
2. PLOT

プロット(最新版)

確定大枠は以下。これが最新であり、それ以前の物語構成(「帰ってきた勇者」「ラスト30分反転」)を置き換える。

  • 過去編: 主人公は調査官として「異変(ほころび=色を失う土地)の調査」から旅を始める。目的は徐々に開示され、やがて「世界が崩壊に向かっている」ことが分かる。ラスボスを倒し、いったん世界の崩壊を防いだ形で終わる
  • どんでん返し: ラスボスは実は世界の均衡を保つ存在だった。均衡には創造だけでなく一定の破壊が必要であり、そのために悪役に見えていた。倒したことで、真の崩壊が始まる
  • 現在編: 2026年の現実の街。上記の真実が明らかになり、真の世界の崩壊を防ぐ物語が続く。過去編のほころび=現在の「消えた場所」(更地・工事フェンス・再開発)として現れる

検討中物語設定書 v2.0(新プロット版)は未執筆。過去編/現在編の時間配分、章構成、ラスボスの名前と正体の演出、現在編の敵・障害の形、エンディング。旧構成の道具(ほのめかし三段階、遅延タイトルドロップ、二重台帳、命名規則)は使える部分を新プロットに移植する。

旧プロット(資産): 「1987年のレトロRPGだと思っていたものが、ラスト30分で老人の実際の記憶だったと反転する」構成(設定書v1.0)。感情装置として強いが「わくわく不足」のため主軸から降格。要素単位(祖父の日記=継承、まぼろし座/かたりべ座、白銅貨の声など)は新プロットのサブクエ・演出に転用候補。

確定現物: 共有グリッド 24×72 (探索ビルド)

南から: はじまりの村/住宅地(家10軒・井戸・畑↔駐車場・柵↔ブロック塀) → 街道/土手の道(祠↔地蔵・ほころび→売地・池↔調整池・立て札↔青標識) → 大きな川/多摩川(石橋↔現代の橋・石積み↔テトラポッド) → 麦畑↔資材置き場 → リーメンの都/団地(城壁・王の塔↔給水塔・家6軒↔団地5棟・あきんどの店↔コンビニ・ほころび→工事フェンス)。ねこが両時代の同じ場所にいる。

5. SYSTEMS

ゲームシステム仕様

  • 確定フィールドと戦闘の分離。戦闘はコマンド式(たたかう/じゅもん/どうぐ/にげる)+AUTOモード(HP35%未満で回復優先の簡易AI・設定保存)
  • 確定移動: 半タイル単位(Ys2式・1歩85ms)。当たり判定は重なる最大4セル。十字キー+Aボタン(タッチ)/矢印+Enter(PC)
  • 確定セーブ: localStorage自動保存+祠でのセーブ演出。全滅ペナルティ軽微(所持金半分で拠点へ)
  • 確定稼ぎ最小の5原則: ①成長の主軸は装備と物語報酬 ②イベント=恒久成長 ③地形で危険が読める ④境界(橋・関所・門)=レベルゲート ⑤再走負担ゼロ(帰還手段・軽ペナルティ)。EXP曲線は「章のルート戦闘だけで足りる」調整
  • 確定敵の出現: 草地/森で歩数判定、道は低率。川や門を境に敵が強くなる(DQ1の橋の文法)
  • 検討中現在編のゲームプレイ: 戦闘があるのか、探索・謎解き中心か。時代切替をパズル/進行の鍵にする設計(過去で変えると現在が変わる等)。新プロットの根幹なので次の設計対象
6. DATA

データ仕様(DQ1級)

  • 確定規模: 呪文10 ／ 武器7・鎧7・盾3 ／ 敵40種(13系統×色違い+ボス)
  • 確定呪文: いやし・いかずち・ねむり・あかり・くちどめ・かえりみち・ぬけみち・まもり・おおいやし・らいめい。半分はレベル、半分は書院の本で習得(『雷のしくみ(図解つき)』等=軽いほのめかし)
  • 確定装備の要点: 店買い3段階+宝箱+ドロップ。最強武器「やくそくの木剣」/最強鎧「ひかりの羽織」はサブクエ報酬(店で買えない思い出の品)。かがみの盾=呪文反射
  • 確定敵の表現ルール: 表の層は正統ファンタジー名のみ(スライム/やまいぬ/じごくいぬ/みかづきのまじょ/井戸の精/地這いの亡者/雷鳴竜…)。「子供の頃こわかったものの翻訳」という正体は図鑑の裏テキストとして終盤に開示
  • 資産サブクエスト6本(約2時間): 白銅貨の声/かたりべ座の返しわすれ/竜骨の秘密基地/消えた霊峰/からくり盤の名前/坂の寺町の蔵(祖父も「帰ってきた」ことがある=継承)。旧プロット文脈で設計→新プロットへ再マップして採用予定
  • 検討中ラスボス(均衡の守り手)の設定・名前。旧「魔王クロガネ(工場の火と煙)」は正体設定を差し替えて続投候補
7. TECH

技術仕様

  • 確定実行環境: 1ファイル完結のHTML+Canvas(JS)。バックエンド不要、セーブはlocalStorage。配信はArtifact(開発試遊)→GitHub Pages(公開)。GASは通信要素(セーブ共有等)が欲しくなった場合の将来枠
  • 確定描画方式: ワールド全体をオフスクリーンに事前焼き込み(過去=2フェーズで波/旗アニメ、現在=1枚)→毎フレームはカメラ切り出し+スプライト合成。ピクセルバッファ(ImageData)+ベイヤーディザ関数がPC-88側の中核
  • 確定スプライト: すべてコード生成(外部画像なし)。過去16px/現在16×24px、2コマ歩行、4方向
  • 確定品質保証: Playwright(内蔵Chromium)による自動通しテスト(移動/会話/セーブ/戦闘/時代切替)+スクリーンショット目視。window.DEBUGフック(warp/era/state)を各ビルドに装備
  • PWA: ホーム画面追加で全画面(apple-mobile-web-app-capable)
